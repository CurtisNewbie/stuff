# 周报模板与示例

## 输出格式说明

周报内容填入团队飞书周报表格，每行对应一个需求/事项，只需填两列：

- **重点项目**：项目名 + 项目级 context（整体进展、关键指标、风险）
- **任务描述**：`（进度%）` + What/Why/When/How

---

## 标准模板

```
【重点项目列】
[项目名]
[项目级 context：本周整体进展、关键指标变化、阻塞点或下周关键节点。如无则只填项目名。]

【任务描述列】
（进度%）
- What：[本质上做了什么能力？解决了什么核心问题？]
- Why：[为什么要做这件事？业务背景或技术驱动因素？]
- When：[关键里程碑 / 完成时间 / 当前进展节点]
- How：[具体的技术方案或操作方法]
```

同一项目下多个需求：**重点项目列合并（rowspan）**，任务描述列每个需求单独一行。

---

## 示例

### 输入（飞书任务数据 + 用户补充）

```
飞书任务链接：
- [任务ID-1]：[需求A 标题]
- [任务ID-2]：[需求B 标题]

用户补充项目级 context：
[项目名] - [关键指标变化，如：核心指标从 X 上升至 Y]。
[阻塞或风险：如推进偏慢、依赖方响应慢、资源不足等，如无则省略。]
```

### 输出

```
【重点项目列 - 行1】
[项目名]
[项目级 context：整体进展 + 关键指标变化 + 阻塞/风险。如无项目级 context 则只填项目名。]

【任务描述列 - 行1（对应需求A）】
（100%）
- What：[本质上做了什么能力？解决了什么核心问题？]
- Why：[业务背景或技术驱动因素——为什么要做这件事？]
- When：MM/DD–MM/DD 完成开发与调试。
- How：[具体技术方案或操作方法；如有可量化效果则附数据，如"耗时从 Xs 降至 Ys"。]

【重点项目列 - 行2（rowspan，与行1合并，不重复填写）】

【任务描述列 - 行2（对应需求B）】
（80%）
- What：[本质上做了什么能力？解决了什么核心问题？]
- Why：[业务背景或技术驱动因素。]
- When：MM/DD 完成，预计 MM/DD 上线。
- How：[具体实现方式；进行中的可说明当前进展节点。]
```

---

## 常见问题

**Q: 进度怎么填？**
已完成填 `（100%）`，进行中填具体比例如 `（80%）`，已上线填 `（已上线）`。

**Q: 项目级 context 从哪里来？**
需要询问用户。飞书任务 API 能拿到单条需求的描述，但项目整体状况（指标变化、阻塞、团队情况）只有用户知道。

**Q: 多个小事项（排查、配置）要合并吗？**
性质相近的可以合并进一个任务描述行，在 How 里列举具体问题。

**Q: 用户只有简短的任务名，3W2H 信息不全怎么办？**
优先询问 Why（最重要）和 When（进展节点）。不要凭空编造 How 的技术细节。

## 重点项目进展更新（风控2026重点事项表）

基于同一份输入，额外生成每个 AI 项目行的「关键进展」列更新。

### 格式

```
MM/DD 进度：
- [关键结果/当前状态]
- [当前阻塞（如有）]
- [接下来]
```

### 示例

```
【[项目A 名称]】
MM/DD 进度：
- [关键结果或当前状态，如：完成方案设计，已开始开发。]
- [当前阻塞（如有），如：依赖方尚未回复，推进受阻。]
- [接下来，如：下周完成联调，预计 MM/DD 上线。]

【[项目B 名称]】
MM/DD 进度：
- [关键结果，如：功能已上线，核心指标从 X 改善至 Y。]
- [接下来，如：持续观察数据，收集用户反馈。]

【[项目C 名称]】
MM/DD 进度：
- [当前状态，如：仍在产品验收中。]
```
