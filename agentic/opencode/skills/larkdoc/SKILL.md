# Lark Document Markdown Style Guidelines

When writing or editing markdown content for 飞书/Lark documents, follow these rules:

1. **No extra horizontal rules (`---`)** — Do not insert `---` dividers in the document. Lark renders these as visual separators that clutter the layout.

2. **No extra blank lines between sections** — Do not add extra blank lines before or after headings. Use exactly one blank line to separate sections, no more.

### Example: Section Spacing

Bad:

```
### Section A

Some content here.


### Section B

More content here.
```

Good:

```
### Section A
Some content here.

### Section B
More content here.
```
